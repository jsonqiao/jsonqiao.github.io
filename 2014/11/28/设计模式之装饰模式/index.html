设计模式之装饰模式

发表于 2014-11-28   |   分类于 设计模式   |   
设计模式是编程经验、是对OO设计准则的良好编程实践。灵活运用是重点，切忌生搬硬套。
装饰模式定义
原文：<span style="font-family: Baskerville-Bold; font-size: 10pt; color: #000000; font-style: normal; font-variant: normal;"><span style="font-family: Baskerville; font-size: 10pt; color: #000000; font-style: normal; font-variant: normal;"><span style="font-size: 12pt;"><span style="font-family: georgia,palatino;">The Decorator Pattern attaches additional </span><span style="font-family: georgia,palatino;">responsibilities to an object dynamically. </span><span style="font-family: georgia,palatino;">Decorators provide a flexible alternative to </span><span style="font-family: georgia,palatino;">subclassing for extending functionality.</span></span>


翻译：装饰模式可以动态为对象绑定额外的职责，为了扩展功能，装饰者提供一种比继承更灵活的替代方案。
装饰模式UML

OO设计准则
  1. 类应该对扩展开发对修改关闭
应用场景
正如前文中装饰模式的定义为类动态赋予额外的职责，所以应用场景主要是当想朝多个方向扩展功能时，该模式能够发挥比较好的功效。
示例代码
package com.jsonqiao.decoratorPattern;

/**
 * 装饰模式的抽象接口
 * Created by json_qiao on 2014/11/24.
 */
public abstract class Ball {

    /**
     * 名称
     */
    public abstract String name();

    /**
     * 用途
     */
    public abstract String usage();
}

package com.jsonqiao.decoratorPattern;

/**
 * 篮球
 *
 * Created by json_qiao on 2014/11/24.
 */
public class Basketball extends Ball {

    @Override
    public String name() {
        return "basketball";
    }

    @Override
    public String usage() {
        return "You can play basketball with me!";
    }
}

package com.jsonqiao.decoratorPattern;

/**
 * 装饰类
 *
 * Created by json_qiao on 2014/11/24.
 */
public abstract class Decorator extends Ball {

    protected Ball ball;

}

package com.jsonqiao.decoratorPattern;

/**
 * 装饰篮球的名称
 *
 * Created by json_qiao on 2014/11/24.
 */
public class NameDecorator extends Decorator {

    public NameDecorator(Ball ball){
        super();
        this.ball = ball;
    }

    @Override
    public String name() {
        return "jsonqiao-" + ball.name();
    }

    @Override
    public String usage() {
        return ball.usage();
    }
}

package com.jsonqiao.decoratorPattern;

/**
 * Created by json_qiao on 2014/11/24.
 */
public class DecoratorPatternTest {

    public static void main(String[] args) {
        Ball basketball = new Basketball();
        System.out.println(basketball.name());
        Decorator decorator = new NameDecorator(basketball);
        System.out.println(decorator.name());
        decorator = new NameDecorator(decorator);
        System.out.println(decorator.name());
    }

}
