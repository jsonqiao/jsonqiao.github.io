CenOS 6.5安装gitlab过程实录

发表于 2015-04-03   |   分类于 Linux   |  
gitlab安装英文说明手册：centos-6-5-install-gitlab
1.升级系统包
升级包： yum -y update
如果没有安装wget 安装wget：yum -y install wget
2.增加EPEL源
增加源：sudo wget -O /etc/pki/rpm-gpg/RPM-GPG-KEY-EPEL-6 https://www.fedoraproject.org/static/0608B895.txt
将源倒入系统：sudo yum --import /etc/pki/rpm-gpg/RPM-GPG-KEY-EPEL-6

确认是否成功：sudo rpm -qa gpg*
输入结果：gpg-pubkey-0608b895-4bd22942
3.安装epel-release-6-8.noarch包
sudo rpm -Uvh http://dl.fedoraproject.org/pub/epel/6/x86_64/epel-release-6-8.noarch.rpm
4.添加PUIAS Computational repository
创建文件：vim /etc/yum.repos.d/PUIAS_6_computational.repo

在该文件中添加如下内容：   
[PUIAS_6_computational]
name=PUIAS computational Base $releasever - $basearch
mirrorlist=http://puias.math.ias.edu/data/puias/computational/$releasever/$basearch/mirrorlist
#baseurl=http://puias.math.ias.edu/data/puias/computational/$releasever/$basearch
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-puias

下载并安装gpg key：
sudo wget -O /etc/pki/rpm-gpg/RPM-GPG-KEY-puias http://springdale.math.ias.edu/data/puias/6/x86_64/os/RPM-GPG-KEY-puias
sudo rpm --import /etc/pki/rpm-gpg/RPM-GPG-KEY-puias

确认安装结果：
sudo yum -qa gpg*
输出：gpg-pubkey-41a40948-4ce19266
5.确认EPEL和PUIAS Computational repositories可用
sudo yum repolist

输入结果如下表示可用：
 repo id               repo name                           status
 PUIAS_6_computational PUIAS computational Base 6 - x86_64 2,018
 base                  CentOS-6 - Base                     4,802
 epel       Extra Packages for Enterprise Linux 6 - x86_64 7,879
 extras                CentOS-6 - Extras                   12
 updates               CentOS-6 - Updates                  814
 repolist: 15,525

官方文档说如果使用如上命令看不到上面红色字体标示的内容，请尝试用sudo yum-config-manager --enable epel --enable PUIAS_6_computational（本人未尝试）
6.安装gitlab所必需的工具和包
切换到root  su
yum -y update
yum -y groupinstall 'Development Tool
yum -y install vim-enhanced readline readline-devel ncurses-devel gdbm-devel glibc-devel tcl-devel openssl-devel curl-devel expat-devel db4-devel byacc sqlite-devel gcc-c++ libyaml libyaml-devel libffi libffi-devel libxml2 libxml2-devel libxslt libxslt-devel libicu libicu-devel system-config-firewall-tui python-devel redis sudo wget crontabs logwatch logrotate perl-Time-HiRes git

注意事项：如果某些包（例如，gdbm-devel, libffi-devel and libicu-devel等）不能成功安装，请使用yum-config-manager --enable rhel-6-server-optional-rpms命令（本人未尝试）
7.配置redis
sudo chkconfig redis on   # 开机启动redis
sudo service redis start  # 启动redis服务
8.邮件服务配置
su   # 切换到root
yum -y install sendmail-cf   # 安装邮件服务
cd /etc/mail && vim /etc/mail/sendmail.mc   # 配置邮件服务文件
    ＃ 修改内容
    添加发件服务器网关：define(`SMART_HOST`, `smtp.yours.com`)dnl
    替换行：EXPOSED_USER(`root`)dnl为dnl EXPOSED_USER(`root`)dnl
9.ruby安装与配置
su   ＃ 切换到root
mkdir /tmp/ruby && cd /tmp/ruby   # 创建目录并进入该目录
curl --progress ftp://ftp.ruby-lang.org/pub/ruby/2.0/ruby-2.0.0-p353.tar.gz | tar xz   # 下载并解压缩
cd ruby-2.0.0-p353
./configure --prefix=/usr/local   # 配置安装路径
make && make install   # 安装

注销在登陆后查看是否安装成功
which ruby
输出：/usr/local/bin/ruby
ruby -v
输出：ruby 2.0.0p353 (2013-11-22 revision 43784) [x86_64-linux]

sudo gem install bundler --no-ri --no-rdoc   # 安装bundler
10.创建gitlab账户
su   # 切换到root
adduser --system --shell /bin/bash --comment 'GitLab' --create-home --home-dir /home/git git   #  创建git用户

# 配置邮件服务
echo yours@email.com > /root/.forward
chown root /root/.forward
chmod 600 /root/.forward
restorecon /root/.forward

echo yours@email.com > /home/git/.forward
chown git /home/git/.forward
chmod 600 /home/git/.forward
restorecon /home/git/.forward
11.GitLab shell 配置

GitLab shell是一个针对GitLab开发的ssh访问和repository管理软件。

su   ＃ 切换到root
su git   # 切换到刚刚创建的git用户
cd ~ && git clone https://github.com/git/gitlabhq/gitlab-shell.git   # 创建git－shell本地仓库
cd gitlab-shell && git checkout v1.8.0   # 切换到版本1.8.0
cp config.yml.example config.yml   # 创建配置文件
vim config.yml   # 编辑该文件
    ＃ 编辑内容为修改gitlab_url,通常配置为http://localhost:8080/
./bin/install   # 进行必要的安装
12.Mysql安装与配置
su   # 切换到root
yum -y install mysql-server mysql-devel   # 安装mysql服务
chkconfig mysqld on   # mysql服务开机自动启动
service mysqld start   # 启动mysql服务

mysql安全配置
/usr/bin/mysql_secure_installation   # 所有问题回答为yes

＃ mysql登陆
create user 'gitlab'@'localhost' identified by 'supersecret';   # 创建用户和密码
create database if not exists `gitlabhq_production` default character set `utf-8` collate `utf8_unicode_ci`;   # 创建gitlab production数据库
grant select, lock tables, insert, update, delete, create, drop, index, alter on `gitlabhq_production`.* to 'gitlab@localhost';   # 分配必要的权限
＃ 尝试用新建的账户登录mysql数据库
mysql -u gitlab -p -D gitlabhq_production
13.GitLab安装与配置
# 将gitlab安装到用户git主目录下
su git && cd ~   # 切换到git主目录
git clone https://github.com/gitlabhq/gitlabhq.git gitlab   # 创建本地仓库
cd gitlab   # 进入gitlab目录
git checkout 6-3-stable   # 切换到该分支
cp config/gitlab.yml.example config/gitlab.yml   # 创建配置
sed -i 's|localhost|your domain|g' config/gitlab.yml   # 修改配置中的域名配置
＃ 增加写的权限给log和tmp目录
chown -R git log/   # 修改目录所有者给git
chown -R git tmp/   # 同上
chmod -R u+rwx log/   # 增加所有者的权限
chmod -R u+rwx tmp/   # 同上

mkdir gitlab-satellites   # 为satellites创建目录

# 为pids和sockets创建目录并赋予相关权限
mkdir tmp/pids && mkdir tmp/sockets
chmod -R u+rwx tmp/pids && chmod -R u+rwx tmp/sockets

mkdir public/uploads && chmod -R u+rwx public/uploads # 增加备份目录以及相关权限

cp config/unicorn.rb.example config/unicore.rb   ＃ 创建unicore配置

＃ 配置git系统账户
git config --global user.name "GitLab"
git config --global user.email "gitlab@your_domain_name" # 此处的email地址与gitlab.yml配置文件中的保持一致
git config --global core.autocrlf input

# 配置GitLab数据库
cp config/databse.yml.mysql config/database.yml   # 创建配置文件
vim config/database.yml   # 修改配置文件
    ＃ 修改内容
    config/databse.yml配置文件中的用户名和密码
chmod o-rwx config/database.yml   # 只读权限给用户git

＃ 安装Gems
su   ＃ 切换到root
gem install charlock_holmes --version '0.6.9.4' # 安装命令
备注：执行如上命令如果出现错误，可能原因是中国防火墙以及其它原因，本人在执行以上命令后出现了一些错误，具体错误记不住了，将修改的内容记下
Gemfile: source "url" 改为 source "http://ruby.taobao.org"
出现问题:Could not find modernizr-2.6.2 in any of the sources修改如下内容Gemfile:modernizr改为modernizr-rails，版本从2.6.2改为2.7.1
Gemfile.lock:modernizr改为modernizr-rails，版本从2.6.2改为2.7.1

修改完成后在执行如上命令即可

cd /home/git/gitlab
bundle install --deployment --without development test postgres puma aws 

# 初始化数据库以及激活特性集
cd /home/git/gitlab
bundle exec rake gitlab:setup RAILS_ENV=production   # 选择yes后系统管理员账户将被创建

＃ admin@local.host/5iveL!fe

＃ 安装初始化脚本
su   # 切换到超级管理员
wget -O /etc/init.d/gitlab https://raw.github.com/gitlabhq/gitlab-recipes/master/init/sysvinit/centos/gitlab-unicorn   # 下载脚本
chmod +x /etc/init.d/gitlab   # 修改权限
chkconfig --add gitlab   #  增加开机启动
chkconfig gitlab on

# 检查GitLib配置是否正确
su git   # 切换到git用户
cd ~/gitlab
bundle exec rake gitlab:env:info RAILS_ENV=production

# 修改下载的初始化脚本文件/etc/init.d/gitlab
  修改内容如下：
             # start unicore注释下配置的路径：bin/web start 改为 script/web start
             # start sidekiq注释下配置的路径：bin/background_jobs start改为script/background_jobs start

service gitlab start   # 启动gitlab

＃ 检查gitlab是否成功运行
su git    # 切换到git用户
cd ~/gitlab
bundle exec rake gitlab:check RAILS_ENV=production

# 输出如下
输出your init script is not up-to-date 回答为no的状态，可被忽略
14.配置nginx服务器
su   # 切换到root
yum -y install nginx   # 安装nginx服务器
chkconfig nginx on   # 开机自动启动

cp /home/git/gitlab/lib/support/nginx/gitlab /etc/nginx/conf.d/   ＃拷贝配置
mv /etc/nginx/conf.d/default.conf /etc/nginx/conf.d/default.conf.back    ＃备份默认配置
# 重盖默认配置(或者删除default.conf 默认配置，只用gitlab) 
mv /etc/nginx/conf.d/gitlab /etc/nginx/conf.d/default.conf

service nginx start   # 启动nginx
15.配置防火墙
lokkit -s http -s https -s ssh   ＃ 通过
service iptables restart   # 重启
