fstab配置说明

发表于 2015-01-08   |   分类于 Linux   |  
fstab

综述：fstab文件表示文件系统的静态信息，位于/etc/fstab, 该文件包含7个字段，下面对每个字段说明。

fstab字段说明

# /etc/fstab: static file system information.
#

<file system> <mount point> <type> <options> <dump> <pass>

<file system>：该字段说明被挂载的文件系统，可用目录树结构和UUID来表示，如下所示：

# 目录树形式
/dev/sda1 / ext4 errors=rem ount-ro 0 1

UUID形式

UUID=536c0787-9c8e-4783-9155-a5e265f0e8b5 / ext4 errors=rem ount-ro 0 1
<mount point>：该字段说明文件系统的挂载点。关于何为挂载点此处不述，可自行google之。

# 文件系统/dev/sda1挂载到根（/）目录
/dev/sda1 / ext4 errors=rem ount-ro 0 1

<type>：该字段说明文件系统的类型。可用mount命令和df -T命令查看文件系统类型，linux常见的文件系统有ext2,ext3,ext4,minix等。

<options>：该字段说明文件系统的挂载选项，多个挂载选项以逗号分隔；

# 常用选项说明
defaults：常用defaults为rw, suid, dev, exec, auto, nouser,and async
     rw：按读写权限挂载
     auto：开机自动挂载
    nouser：超级用户挂载

noauto：不允许开机自动挂载或者使用mount -a命令时不挂载
user ：用户挂载
owner ：允许设备拥有者挂载
nofail：当设备不存在时不报告错误信息
<dump>：该字段说明文件系统是否需要备份，当该字段不存在或者该字段值为0时，dump假定该文件系统不需要备份。

<pass>：该字段用于fsck(8)程序在重启时检查文件系统的顺序，根文件系统需要被指定为1，其它文件系统设置2以上。
