发表于 2015-01-21   |   分类于 Linux   |  
全局变量

全局变量或者称为环境变量，这些变量在所有的shell脚本中可用，可用命令 env,printenv显示当前的环境变量。

# 本人当前的部分环境变量
     jsonqiao@jsonqiao:~/shell$ printenv
     XDG_VTNR=7
     LC_PAPER=zh_CN.UTF-8
     SSH_AGENT_PID=1706
     LC_ADDRESS=zh_CN.UTF-8
     XDG_SESSION_ID=c2
     XDG_GREETER_DATA_DIR=/var/lib/lightdm-data/jsonqiao
     SELINUX_INIT=YES
     LC_MONETARY=zh_CN.UTF-8
     CLUTTER_IM_MODULE=xim
     GPG_AGENT_INFO=/run/user/1000/keyring-MSeF75/gpg:0:1
     TERM=xterm
     VTE_VERSION=3409
     SHELL=/bin/bash
     SSH_AGENT_LAUNCHER=upstart
     WINDOWID=73402314
     LC_NUMERIC=zh_CN.UTF-8
     UPSTART_SESSION=unix:abstract=/com/ubuntu/upstart-session/1000/1597
     GNOME_KEYRING_CONTROL=/run/user/1000/keyring-MSeF75
     GTK_MODULES=overlay-scrollbar:unity-gtk-module
     USER=jsonqiao
变量赋值注意点

bash shell中在给变量赋值时不能像java，c++等语言，在=号的左右两侧留有空白，同时能够将变量值用双引号引起来是比较好的编程习惯。

bash特殊变量

# 特殊变量解释说明

$ ： 表示脚本执行的参数，如果用双引号引用该变量，则所有的参数会被解释为单个单词。
$@ ： 表示脚本执行的参数，如果用双引号引用该变量，所有的参数会被解释为一个参数数组。没有双引号时，$和$@作用一致。
$# : 表示脚本的参数个数。
$? : 表示上一个命令执行结束的状态
$$ : 表示执行shell的进程ID。
$! : 表示最近执行的后台任务的进程ID。
$0 : 表示脚本名称。
$- : A hyphen expands to the current option flags as specified upon invocation, by the set built-in command, or those set by the shell itself (such as the -i).
$_ : The underscore variable is set at shell startup and contains the absolute file name of the shell or script being executed as passed in the argument list. Subsequently, it expands to the last argument to the previous command, after expansion. It is also set to the full pathname of each command executed and placed in the environment exported to that command. When checking mail, this parameter holds the name of the mail file.
