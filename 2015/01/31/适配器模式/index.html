设计模式是编程经验、是对OO设计准则的良好编程实践。灵活运用是重点，切忌生搬硬套。
适配器模式定义

原文：The Adapter Pattern converts the interface of a class into another interface the clients expect. Adapter lets classes work together that couldn’t otherwise because of incompatible interfaces.

翻译：适配器模式将一个类的接口转化为客户端期望的接口，适配器让不兼容的接口类能够协同工作。

适配器模式UML

适配器模式应用

适配器模式主要将客户端与变化的接口分离，接口的改变对客户端来说是透明的，这种变化都被封装到适配器中。

示例代码

package com.json.adapterPattern;

/**

  • Created by jsonqiao on 2015/1/31
    */
    public class AdapterClient {
    public static void main(String[] args) {
    Duck mallardDuck = new MallardDuck();
    Turkey wildTurkey = new WildTurkey();

    System.out.println(“calling the turkey”);
    wildTurkey.fly();
    wildTurkey.gobble();

    System.out.println(“原生的duck….”);
    testDuck(mallardDuck);
    System.out.println(“被适配成duck的duck….”);
    testDuck(new TurkeyAdapter(wildTurkey));
    }

    /**

  • 对于该方法，该方法只接收Duck类型的接口，但我们通过适配器
  • 将Turkey类型转化为Duck类型的接口，但对该方法来说是透明的
  • @param duck
    */
    public static void testDuck(Duck duck){
    duck.fly();
    duck.quack();
    }
    }

    public class TurkeyAdapter implements Duck {

    private Turkey turkey;

    public TurkeyAdapter(Turkey turkey){

    this.turkey = turkey;

    }

    @Override
    public void fly() {

    for (int i = 0; i &lt; 5; i++){
    turkey.fly();
}

    }

    @Override
    public void quack() {

    turkey.gobble();

    }
    }
