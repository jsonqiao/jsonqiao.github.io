Java中关于noSuchMethod错误

发表于 2015-05-10   |   分类于 J2EE开发错误   |  
错误出现地方

java.lang.NoSuchMethodError一般出现在maven打包后部署时候（针对用maven管理），在开发阶段由于开发工具比较智能的原因，可能上述问题会被开发工具智能识别而自动屏蔽。

错误原因

出现此类错误的根本原因是jar包存在不同的版本，classloader加载了不正确版本的class文件，一般是加载了低版本的class，所以在你的项目中肯定出现了不同版本的jar包依赖。

解决方法

根据java.lang.NoSuchMethodError后面所跟的报错的类定位jar包，去web应用下WEB-INF/lib目录下找寻对应jar包，一般都会发现不同版本的jar，删除错误的版本。

举例演示

java.lang.NoSuchMethodError: org.apache.velocity.app.VelocityEngine.resourceExists(Ljava/lang/String;)

例如项目部署时出现如上错误，又java.lang.NoSuchMethodError后面的提示信息可知是velocity-tools.jar包中的类报错，于是去项目WEB-INF/lib目录下找寻相关的jar即可，找到后删除错误的版本。
